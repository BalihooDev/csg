Code Style Guide


To provide Balihoo's front-end developers a core coding style guide to create more uniform standard code.

HTML

Spacing

Spacing in standard markup will indent by one tab (or four spaces) and have one line space between each line of code.

<div>

 →       <p>Hello World</p>

 . . . . <p>Hello World</p>

</div>

Comments

Comments will fall into three categories: inline, closing, and block. Inline comments are for on-the-fly labeling, reminders, and other short notes. Closing comments come in handy for keeping track on multiple nested elements, especially block elements like divs. Proper usage of the closing comment should be almost an extension of the closing tag and include the elements selector(e.g. <!-- /wrapper -->). Avoid commenting on obvious code.

Inline Comment:

<p>Hello World</p><!-- Inline Comment -->

Closing Comment:

<div id="wrapper">

    <p>Some text</p>

    <p>Some more text</p>

</div><!-- /wrapper -->

Block Comment:

<!-- \/ Block Comment Start -->
<div>

    <p>Some text</p>

    <p>Some more text</p>

</div>
<!-- /\ Block Comment End -->

CSS

Spacing, Naming Convention, and Comments

For the stylesheet spacing, insert one line space between each declared selector. Each style will be tabbed once(four space) and have a line break after the value. Aligning a style's value is optional? Use ANSI style brackets and camel case for the selectors(e.g. myClass). For comments we will have three types: inline, descriptive and block.

Spacing & Naming Convention

.firstClass {
→       margin: 0;
. . . . padding: 0;
}

.secondClass {
→       margin: 0;
. . . . padding: 0;
}

Comments:

.firstClass {
    margin: 0; /* Inline Comment */
}

.secondClass {
    /*
     * This is a bunch of descriptive
     * text that explains something
     * that applies to this selector.
    */
    margin: 0;
}

/* \/ -- Code Section Start -- */
.thirdClass {
    margin: 0;
}
/* /\ -- Code Section End -- */ (optional end cap)

Stylesheet Structure

/*
 * Title: Stylesheet Name
 * Created by: Author
 * Date Created: Month, Year
 * Desription: Brief description
*/

/* \/ -- Resets -- */
body {
    margin:  0;
    padding: 0;
}

/* \/ -- Global Typography -- */
h1, h2, h3... {
    margin:  0;
    padding: 0;
}

/* \/ -- Global Elements (Borders, Arrows, Etc.) -- */
.dottedBorder{
    border: 1px dotted white;
}

Javascript & jQuery

Functions

For both javascript and jQuery use ANSI style brackets and spacing. Indent one tab (four spaces) and add a line space between lines of code. Name functions in camel case (e.g. myFunction).

function( arg ){

// your logic here

};

Variables

When creating local scope variables in both JS and jQuery using var if there is more than one variable a comma should be used (except for the last variable which ends with a semi-colon) and all subsequent variables should be aligned with the first variable.

var creativeDevs,
    scott,
    djame,
    becca;

jQuery Methods

If a project calls for a custom jQuery method to be created follow the standard below using the same styles and indentation as the functions, just within the scope of the jQuery object.

$.fn.extend({


 →       myFirstMethod: function (){

 . . . . // Method Logic Here

    }

 . . . . mySecondMethod: function (){

 →       // Method Logic Here

    }

});
